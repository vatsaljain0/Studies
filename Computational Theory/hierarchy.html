The Chomsky Hierarchy

Exam Edition: Simple Explanations, Rules, and Machines

1. The Big Picture (Concentric Circles)

Proposed by Noam Chomsky, this hierarchy classifies formal languages into 4 levels. Every language on the inside is automatically a part of the languages on the outside. (e.g., Every Regular Language is also Context-Free).

💡 Memory Trick: The Stricter You Are, The Smaller Your Circle

Type 3 (Regular) is in the very center because it has the strictest rules. Type 0 is on the outside because it is the "God mode" level with zero restrictions. As you go from 3 down to 0, the machines get more powerful, and the rules get looser.

2. The 4 Grammar Types (Simple English)

Type 0: Unrestricted Grammar (The Boss)

This is the highest level. It generates Recursively Enumerable languages. Because it is "unrestricted," there are practically no rules on what the productions can look like.

Rule: α → β
(α must contain at least one non-terminal/uppercase letter. β can be literally anything).
  • Accepted by: Turing Machine (TM).
  • Example: `aB c → d e F` (Totally fine! Multiple symbols on the left, anything on the right).

Type 1: Context-Sensitive Grammar (The Weight-Watcher)

Generates Context-Sensitive Languages. The only major rule here is that the right side of the arrow must never be shorter than the left side. It cannot shrink!

Rule: α → β
(Condition: The length of |α| ≤ the length of |β|)
  • Accepted by: Linear Bounded Automaton (LBA).
  • Important Note: Because it cannot shrink, ε (epsilon, which has length 0) is usually not allowed on the right side.
  • Example: `A B → a B c` (Valid: Left side has 2 symbols, right side has 3. Length increased!)

Type 2: Context-Free Grammar (The Lone Wolf)

Generates Context-Free Languages. This is what you use when simplifying grammars! The strict rule here is that the left side of the arrow MUST be exactly one single uppercase Variable. Nothing else.

Rule: A → γ
('A' is exactly ONE non-terminal. 'γ' can be any combination of terminals and non-terminals).
  • Accepted by: Pushdown Automaton (PDA).
  • Example: `S → a S b` (Valid: Left side is exactly one uppercase letter).
  • Invalid Example: `a S → a b` (Invalid for Type 2: Left side has a terminal 'a' next to the 'S').

Type 3: Regular Grammar (The Strictest Parent)

Generates Regular Languages. This is the most restrictive level. Like Type 2, the left side must be one uppercase variable. But now, the right side is strictly controlled: it can only have at most one terminal (lowercase), which can optionally be followed by exactly one variable (uppercase).

Rule (Right Linear): A → aB   OR   A → a
('A' and 'B' are single Non-Terminals. 'a' is a single Terminal).
  • Accepted by: Finite Automata (DFA / NFA).
  • Example: `S → a B` or `S → a`.
  • Invalid Example: `S → a S b` (Invalid for Type 3: You cannot have terminals on both sides of a variable).
