DBMS Module 3: Structured Query Language (SQL)

1. Overview of SQL

  • What is SQL: Structured Query Language is used to manage relational databases. It allows users to create, modify, and retrieve data. It strongly follows the ACID properties for transactions.
  • History: Developed by IBM (originally called Sequel) in the early 1970s. Standardized by ANSI in 1986.

Parts of the SQL Language:

  • Data-Definition Language (DDL): Defines database structure (CREATE, ALTER, DROP).
  • Data-Manipulation Language (DML): Modifies database data (SELECT, INSERT, UPDATE, DELETE).
  • Integrity Constraints: Rules the data must satisfy (e.g., Primary Keys).
  • Transaction Control Commands (TCL): Manages transactions (COMMIT, ROLLBACK).
  • Data Control Language (DCL): Manages permissions (GRANT, REVOKE).

2. Data Definition Commands (DDL)

DDL is used to define or modify the actual structure (schema) of the tables, not the data inside them.

  • char(n): Fixed-length string.
  • varchar(n): Variable-length string (saves space if text is short).
  • int: Integer numbers.
  • numeric(p, d): Number with p total digits and d decimal places.
CREATE TABLE Example
Syntax CREATE TABLE table_name (
    column1 datatype constraint,
    column2 datatype constraint
);

Let's create a simple Students table where the Roll Number is the primary key.

CREATE TABLE Students ( RollNo int, Name varchar(50) not null, Age int, primary key (RollNo) );

3. Data Manipulation Commands (DML)

DML is used to add, remove, change, or view the actual data (rows) inside the tables.

INSERT, UPDATE, DELETE, and SELECT
Insert Syntax INSERT INTO table_name (column1, column2) VALUES (value1, value2);

Update Syntax UPDATE table_name SET column1 = value1 WHERE condition;

Delete Syntax DELETE FROM table_name WHERE condition;

Select Syntax SELECT column1, column2 FROM table_name WHERE condition;

Executing these commands sequentially:

-- 1. Add a new student INSERT INTO Students (RollNo, Name, Age) VALUES (101, 'Vatsal', 20); INSERT INTO Students (RollNo, Name, Age) VALUES (102, 'Rahul', 19); -- 2. Update Vatsal's age to 21 UPDATE Students SET Age = 21 WHERE Name = 'Vatsal'; -- 3. Delete Rahul from the table DELETE FROM Students WHERE RollNo = 102; -- 4. View the final table SELECT * FROM Students;
Final Output Table (From SELECT)
RollNoNameAge
101Vatsal21

4. String Operations (The LIKE Operator)

When searching for text, you don't always know the exact word. SQL uses the LIKE operator along with Wildcards to search for patterns.

Understanding % and _ Wildcards
Syntax SELECT column_name FROM table_name WHERE column_name LIKE 'pattern';

1. The Percent Sign (%)

The % symbol represents zero, one, or multiple characters. Think of it as meaning "anything can go here, no matter how long."

• LIKE 'A%' : Matches any word starting with 'A' (e.g., 'Apple', 'Adam', 'A').
• LIKE '%son' : Matches any word ending with 'son' (e.g., 'Johnson', 'Tyson').
• LIKE '%cat%' : Matches any word that contains 'cat' anywhere inside it (e.g., 'Education', 'cats', 'scatter').

2. The Underscore (_)

The _ symbol represents exactly one single character. It acts like a strict placeholder.

• LIKE '_a%' : The second letter MUST be 'a' (e.g., 'Bat', 'Cat', 'Harry').
• LIKE 'S___' : Must start with 'S' and have exactly 3 characters after it (matches 'Sammy', but NOT 'Sam').

Sample Table: `Employees`

NameJob
SarahManager
SamClerk
JasonSales
-- Find anyone whose name contains 'sa' anywhere in it. SELECT Name FROM Employees WHERE Name LIKE '%sa%';
Output
Name
Sarah
Jason
-- Find names that are EXACTLY 3 letters long and start with S. SELECT Name FROM Employees WHERE Name LIKE 'S__';
Output
Name
Sam

Note on ESCAPE: What if you literally want to search for the "%" symbol in text (like "100%")? You use an escape character to tell SQL to treat it as normal text, not a wildcard: LIKE '%100\%%' ESCAPE '\'

5. Set Operations & Null Values

Set operations combine the results of two completely separate SELECT queries into one list. By default, they remove duplicates.

UNION, INTERSECT, EXCEPT
Syntax SELECT column FROM table1
[UNION | INTERSECT | EXCEPT]
SELECT column FROM table2;

Simple Context:

  • Math_Class: Alice, Bob, Charlie
  • Science_Class: Bob, Charlie, David
-- Find students who take BOTH Math AND Science (INTERSECT) SELECT Name FROM Math_Class INTERSECT SELECT Name FROM Science_Class; -- OUTPUT: Bob, Charlie -- Find students taking Math BUT NOT Science (EXCEPT) SELECT Name FROM Math_Class EXCEPT SELECT Name FROM Science_Class; -- OUTPUT: Alice

6. Aggregates, GROUP BY & HAVING

Aggregate functions perform a calculation on a set of values and return a single value (e.g., SUM, AVG, COUNT, MAX, MIN).

GROUP BY groups rows that have the same values into summary rows. HAVING is used to filter those groups (since you cannot use WHERE with aggregate functions).

GROUP BY and HAVING Example
Syntax SELECT column, AGGREGATE_FUNC(column)
FROM table
GROUP BY column
HAVING AGGREGATE_FUNC(column) condition;

Sample Table: `Sales`

RegionAmount
North100
North150
South50
South40
-- Find regions where the TOTAL sales are greater than 100. SELECT Region, SUM(Amount) AS Total_Sales FROM Sales GROUP BY Region HAVING SUM(Amount) > 100;
Output
RegionTotal_Sales
North250

South is excluded because its sum (90) is not greater than 100.

7. Nested Subqueries

A subquery is a query nested inside another query. The inner query runs first, and gives its result to the outer query.

Subquery Example using IN, SOME, ALL
Syntax SELECT column FROM table1
WHERE column operator (SELECT column FROM table2 WHERE ...);

Sample Table: `Products`

ItemPrice
Pen10
Book50
Bag500
-- Find products that cost MORE than the average price of all products. -- Step 1: The subquery calculates the average price (which is 560/3 = 186.6) -- Step 2: The outer query finds items > 186.6 SELECT Item FROM Products WHERE Price > (SELECT AVG(Price) FROM Products);
Output
Item
Bag

SOME vs ALL:

• > SOME means "Greater than at least one item in the subquery list."
• > ALL means "Greater than every single item in the subquery list."

8. SQL Joins

A JOIN is used to combine rows from two or more tables, based on a related column (usually Primary Key and Foreign Key) between them.

[Image of SQL Joins Venn Diagram]
  • INNER JOIN: Returns records that have matching values in BOTH tables.
  • LEFT JOIN: Returns ALL records from the left table, and the matched records from the right table (fills with NULL if no match).
  • RIGHT JOIN: Returns ALL records from the right table, and matched records from the left.
  • FULL JOIN: Returns ALL records from both tables.
INNER JOIN Example
Syntax SELECT table1.col, table2.col
FROM table1
INNER JOIN table2
ON table1.common_field = table2.common_field;

-- Get customer names and the products they ordered SELECT Customers.Name, Orders.Product FROM Customers INNER JOIN Orders ON Customers.CustID = Orders.CustID;

9. Views, Triggers & PL/SQL

A. Views

A view is a "virtual table" based on the result-set of an SQL statement. It doesn't store data itself, but looks like a real table. Used for security (hiding sensitive columns).

View Example
Syntax CREATE VIEW view_name AS SELECT column1, column2 FROM table_name WHERE condition;
-- Create a view that shows employees but HIDES their salary CREATE VIEW Public_Employee_List AS SELECT EmpID, Name, Department FROM Employees; -- Now query it like a normal table: SELECT * FROM Public_Employee_List;

B. Triggers

A trigger is code that automatically runs (fires) just BEFORE or AFTER an INSERT, UPDATE, or DELETE happens on a table. It is used to enforce complex rules.

Trigger Example
Syntax Concept CREATE TRIGGER trigger_name
[BEFORE | AFTER] [INSERT | UPDATE | DELETE] ON table_name
FOR EACH ROW
BEGIN ... END;
-- A trigger that prevents negative salaries from being inserted CREATE TRIGGER Check_Salary BEFORE INSERT ON Employees FOR EACH ROW BEGIN IF NEW.Salary < 0 THEN SIGNAL SQLSTATE '45000' SET MESSAGE_TEXT = 'Salary cannot be negative!'; END IF; END;

C. PL/SQL Block Structure

PL/SQL adds programming logic (like IF statements and loops) to SQL. A Block has 4 sections.

PL/SQL Block Structure
DECLARE -- 1. Declare variables here (Optional) v_message varchar(20) := 'Hello World'; BEGIN -- 2. Write your main logic and SQL queries here (Mandatory) DBMS_OUTPUT.PUT_LINE(v_message); EXCEPTION -- 3. Catch errors/crashes here (Optional) WHEN OTHERS THEN DBMS_OUTPUT.PUT_LINE('An error occurred'); END; -- 4. Marks the end of the block (Mandatory)
