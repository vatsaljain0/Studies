Comprehensive EER Diagrams

Fixed-size rendering mapping 100% of the concepts from Module 1 & 2.
Both diagrams below are standalone examples containing every required EER concept and their relational mappings.

Example 1: University Database EER Diagram

Mapping to Relational Schema (Module 2 Rules)

PERSON (SSN, FName, LName)

PERSON_PHONE (SSN, Phone)

STUDENT (SSN, GPA)

INSTRUCTOR (SSN, Rank)

COURSE (CID, CName)

TAKES (SSN, CID)

SOFTWARE (SoftID, SName)

REQUIRES (SSN, CID, SoftID)

DEPENDENT (SSN, DName, Relation)

* Note: The derived attribute 'Age' is omitted from the physical schema as it is dynamically computed.

Example 2: Corporate Database EER Diagram

Mapping to Relational Schema (Module 2 Rules)

EMPLOYEE (EmpID, City, Zip)

EMP_SKILL (EmpID, Skill)

MANAGER (EmpID)

ENGINEER (EmpID)

PROJECT (ProjID, Title)

WORKS_ON (EmpID, ProjID)

CLIENT (ClientID, CName)

SPONSORS (EmpID, ProjID, ClientID)

* Note: SPONSORS takes the Primary Keys of the Aggregation block (WORKS_ON's keys: EmpID, ProjID) and the CLIENT block (ClientID).
