1. Open-Source and Proprietary Operating Systems

Open-Source OS

Definition: Source code is openly accessible on the internet. Anyone can view, modify, add features, and redistribute it without cost. Developed and tested through open collaboration.

  • Characteristics: Free of cost, customizable, managed by open-source communities.
  • Pros: High flexibility, freedom to modify, no vendor lock-in, rapid security fixes via community scrutiny.
  • Cons: May require technical expertise, less polished UI, limited formal support.
  • Examples: Linux (Ubuntu, Fedora), Android.

Proprietary OS

Definition: Source code is publicly unavailable (confidential). Only the creating company can modify it. Developed and tested by a closed team.

  • Characteristics: Paid software, strictly licensed. Licenses restrict installations, sharing, duration, and features.
  • Pros: Highly polished, professional commercial support, ease of use, extensive hardware compatibility.
  • Cons: Expensive, inflexible, vendor lock-in, users cannot customize core code.
  • Examples: Microsoft Windows, Apple macOS, iOS, iPadOS.

2. Fundamentals of Distributed Operating System (DOS)

Definition: A distributed system is a collection of loosely coupled nodes (independent computers) interconnected by a communication network. A Distributed Operating System manages these physically separate computers, presenting them to users as a single, cohesive system.

DOS Concepts and Fundamentals

  • Transparency: Hides the complexity and physical distribution of resources from users and applications, making the network appear as a single computer.
  • Resource Sharing: Enables efficient sharing of hardware (CPUs, disks) and software (files, databases) across nodes.
  • Scalability: The capability to add more nodes to the system without significant performance degradation or system redesign.
  • Fault Tolerance: The ability of the system to continue functioning correctly when individual components or nodes fail.
  • Heterogeneity: The capacity to manage varied hardware, operating systems, and network protocols within a single system.
  • Concurrency: Multiple processes across different nodes can operate simultaneously, requiring careful synchronization to maintain data consistency.

DOS Architectural Components & Advantages

Architectural Components:
  • Nodes: Independent computers connected by a LAN or WAN.
  • Middleware: A software layer residing between the OS and applications, managing resource allocation.
  • Protocols: Rules governing data exchange (e.g., TCP/IP, RPC).
Advantages:
  • Performance: Parallel processing allows large tasks to be broken down, improving speed.
  • Availability: If one node fails, others take over.
  • Flexibility: Modularity facilitates easy upgrading.

Migration Management in DOS

Data and process migration from one site to another is completely under the control of the distributed OS.

  • 1. Data Migration: Moving data for system upgrades, consolidation, or adopting new technologies.
  • 2. Computation Migration: Transferring the computation (the execution itself), rather than the heavy data, across the system.
  • 3. Process Migration: Executing parts of, or an entire process, at different sites. Used for:
    • Load Balancing: Distributing processes to even the workload.
    • Computation Speedup: Running sub-processes concurrently on different nodes.
    • Hardware Preference: Utilizing specialized processors (like a GPU for matrix inversion).
    • Software Preference: Running the process where specific licensed software physically resides.
    • Data Access: Running the process remotely on the server hosting a massive database, rather than transferring all the data locally.

3. Network Operating System & Cloud Storage

Definition: A Network Operating System (NOS) provides an environment where users can access remote resources (implementing resource sharing) by either logging into the appropriate remote machine or transferring data from the remote machine to their own. (Note: Currently, all general-purpose OSs like Android and iOS function as network OSs).

Remote Access Protocols

Remote Login (SSH)

The internet provides the ssh facility for remote login. A user must have a valid account on the target machine.

Example: ssh username@kristen.cs.yale.edu

Remote File Transfer (FTP)

Allows copying files between machines that maintain their own separate local file systems. Communication is one-directional.

  • get: Transfer file from remote to local.
  • put: Transfer file from local to remote.
  • ls / dir: List files in remote directory.
  • cd: Change remote directory.

Cloud Storage & The "Paradigm Shift"

Cloud-based storage (like Dropbox and Google Drive) allows users to transfer files much like FTP, but via a graphical interface/web link.

Exam Note: The Paradigm Shift

An important point about SSH, FTP, and Cloud storage is that they require the user to change paradigms. FTP requires knowing a command set different from normal OS commands. With SSH, a Windows user connecting to a UNIX machine must abandon Windows commands and switch entirely to UNIX commands for the duration of the session.

2. Core Functions of Cloud OS

  • Resource Virtualization: Abstracts physical hardware (CPUs, RAM, Storage) into virtual resources that can be dynamically assigned to users.
  • Dynamic Provisioning & Scaling: Automatically allocates additional servers or storage when user demand spikes, and scales down when idle.
  • Multi-tenancy Management: Secures and isolates data so multiple distinct users (tenants) can share the same physical server without seeing each other's data.
  • Metering & Billing: Tracks exactly how much compute power, storage, and bandwidth a user consumes for "pay-as-you-go" billing.

B. Real-Time Operating System (RTOS)

An RTOS is an operating system intended to serve real-time applications that process data as it comes in, typically without buffering delays. Its primary goal is not high throughput, but strict deterministic timing and rapid, guaranteed response times.

1. RTOS Architecture

  • Microkernel Architecture: RTOS heavily relies on a microkernel design to keep the core OS footprint tiny. This ensures high stability and rapid execution of core functions.
  • Preemptive Priority-Based Scheduler: A critical component. If a high-priority task enters the ready queue, the RTOS will instantly preempt (interrupt) the current task, regardless of what it is doing.
  • Timer Management: Features high-resolution hardware timers to trigger events and track exact deadlines down to the microsecond.

2. Core Functions of RTOS

  • Deterministic Execution: Guarantees that a specific task will complete within a predefined maximum time limit (deadline).
  • Minimal Interrupt Latency: The time taken from the moment a hardware interrupt occurs to the moment the task starts executing is minimized to near zero.
  • Types of RTOS:
    • Hard Real-Time: Missing a deadline results in total system failure or catastrophic consequences (e.g., Anti-lock brakes, Pacemakers, Missile guidance).
    • Soft Real-Time: Missing a deadline degrades performance but the system can still recover (e.g., Live video streaming, Online gaming).

C. Mobile Operating System

A specialized OS designed explicitly to operate smartphones, tablets, and wearable devices (e.g., Android, iOS). They differ from desktop OSs by being heavily optimized for battery constraints, wireless communication, and touchscreen interfaces.

1. Mobile OS Architecture (Layered Model)

  • Kernel Layer: The lowest level (e.g., Linux for Android, Darwin/XNU for iOS). Handles fundamental hardware interactions, memory management, and process execution.
  • Middleware & Libraries: Sits above the kernel. Contains core libraries like SQLite for databases, WebKit for rendering web pages, and OpenGL for 3D graphics.
  • Application Framework: Provides the APIs (Application Programming Interfaces) that developers use to write apps, like Location Manager or Telephony Manager.
  • Application Layer: The topmost layer containing built-in apps (Dialer, Contacts, Browser) and third-party apps downloaded by the user.

2. Core Functions of Mobile OS

  • Aggressive Power Management: Aggressively puts the CPU, screen, and background apps to sleep to conserve limited battery life.
  • Telephony & Radio Management: Seamlessly manages multiple antennas (Cellular 4G/5G, Wi-Fi, Bluetooth, NFC) and handles active handoffs between networks.
  • Touch & Sensor Processing: Accurately processes multi-touch gestures and interprets data from built-in sensors (Accelerometers, Gyroscopes, GPS).
  • Application Sandboxing: Provides strict security boundaries. Every app runs in its own isolated environment (sandbox) so a malicious app cannot access data from another app without explicit user permission.
