Operating System: Module 2 (Process Management)

1. Basic Concepts of Process & State Transition

The Process: A process is a program in execution. While a program is a passive entity (a file stored on disk), a process is an active entity. It needs resources (CPU time, memory, files, I/O devices) to accomplish its task. The status of its current activity is represented by the program counter and processor registers.

Memory Layout of a Process:

  • Text section: The executable compiled code.
  • Data section: Global and static variables (initialized and uninitialized) that have a fixed size.
  • Heap section: Memory dynamically allocated during program run time (e.g., using malloc() in C or new in Java). Grows upward.
  • Stack section: Temporary data storage when invoking functions (function parameters, return addresses, local variables). Grows downward toward the heap.

The 7-State Process Transition Model:

A process goes through different states during its lifecycle. While basic models show 5 states, the complete advanced model includes 7 states to account for memory swapping (virtual memory management):

  • 1. New: The process is being created and admitted to the system.
  • 2. Ready: The process is loaded into main memory and is waiting to be assigned to a processor by the Short-Term Scheduler.
  • 3. Running: Instructions are actively being executed (CPU is allocated).
  • 4. Waiting (Blocked): The process is waiting for some event to occur (e.g., waiting for user input, file I/O completion).
  • 5. Terminated: The process has finished execution or is forcefully terminated.
  • 6. Suspended Ready: If main memory is full, a Ready process is swapped out to secondary storage (disk) to free up RAM. It is ready to run but not in main memory.
  • 7. Suspended Blocked (Suspended Wait): A blocked/waiting process is swapped out to disk. If the I/O event completes while it is on disk, it transitions to Suspended Ready.

2. Operations on Processes & PCB

Process Creation:

  • A process (parent) may create several new processes (children), forming a tree of processes.
  • Identified by a unique process identifier (pid). (Example: Linux systemd process has pid = 1, serving as the root parent process for all user processes).
  • UNIX Example: The fork() system call creates a child process (child pid=0, parent gets child's actual pid). The child inherits privileges/open files. The child overlays its address space using exec(). The parent waits using wait() and resumes when the child invokes exit().

Process Termination:

  • A process terminates when it finishes its final statement and asks the OS to delete it using the exit() system call. It returns a status value to its waiting parent via wait().
  • All resources (physical/virtual memory, open files, I/O buffers) are deallocated by the OS.
  • Cascading Termination: If a parent process terminates, the OS often does not allow its child processes to continue, forcefully terminating the entire subtree.

3. Context Switching

Definition: The process where the CPU stops running one process, saves its current state in its PCB, and loads the saved state of another process so that multiple processes can share the CPU effectively.

  • Role in Scheduling: The scheduler decides the next process; the dispatcher performs the actual switch.
  • Overhead: Time spent switching is called context switch overhead. The CPU does no useful work during this time. If context switching is too frequent, system performance drastically drops.

4. Threads & Process Differences

Thread: Often called a "lightweight process." It is a basic unit of CPU utilization comprising a Thread ID, Program Counter (PC), Register set, and Stack. Multiple threads within the same process share the code section, data section, and OS resources (like open files).

Process vs. Thread (Key Differences):

Process Thread
Heavyweight: Takes more time to create, terminate, and manage. Lightweight: Faster to create, terminate, and manage.
Isolated Memory: Each process has its own isolated memory address space. Shared Memory: Threads within a process share the same memory (Code, Data, Heap) but have their own Stack.
Context Switching: High overhead. OS must swap out entire memory mappings and PCBs. Context Switching: Low overhead. No need to switch memory maps.
Communication: Requires slow Inter-Process Communication (IPC) like Pipes or Message Passing. Communication: Fast and direct through shared memory variables.
Fault Isolation: If one process crashes, it does not affect other processes. Fault Vulnerability: If one thread crashes, the entire process (and all its threads) can crash.

Multithreading Models:

  • Many-to-One: Maps many User-Level Threads to one Kernel-Level Thread. Entire process blocks if one thread blocks.
  • One-to-One: Maps each User Thread to a Kernel Thread. Maximum concurrency, but high OS overhead (used by Windows, Linux).
  • Many-to-Many: Multiplexes many User Threads to a smaller/equal number of Kernel Threads.

5. Basic Concepts of Scheduling & Types of Schedulers

  • 1. Long-Term Scheduler (Job Scheduler): Loads processes from disk into main memory. Controls the degree of multiprogramming.
  • 2. Short-Term Scheduler (CPU Scheduler): Selects a process from the ready queue and assigns the CPU. Operates extremely fast (every few milliseconds).
  • 3. Medium-Term Scheduler: Manages swapping. It temporarily removes processes from main memory to disk (Swap out to Suspended Blocked/Ready states) to free up space, and brings them back later (Swap in). Reduces the degree of multiprogramming.

6. Scheduling Algorithms & Examples

A. Definitions & Formulas

  • Non-Preemptive: A process holds the CPU until it terminates or waits. It cannot be interrupted.
  • Preemptive: OS can interrupt a running process and force it into the ready state to allocate the CPU to a higher priority process.
  • Turnaround Time (TAT) = Completion Time (CT) - Arrival Time (AT)
  • Waiting Time (WT) = Turnaround Time (TAT) - Burst Time (BT)

1. First-Come, First-Served (FCFS) [Non-Preemptive]

Managed with a FIFO queue. The process that arrives first gets the CPU first. It suffers from the Convoy Effect (short processes forced to wait behind long processes).

D. Multilevel Queue Scheduling

With both priority and round-robin scheduling, all processes may be placed in a single queue, requiring an O(n) search to determine the highest-priority process. In practice, it is easier to have separate queues for each distinct priority.

Priority scheduling simply schedules the process in the highest-priority queue. If there are multiple processes in the highest-priority queue, they are executed in Round-Robin order. In its most generalized form, a priority is assigned statically to each process, and it remains in the same queue for the duration of its runtime.
