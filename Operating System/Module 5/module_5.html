File Systems & Disk Scheduling

Comprehensive Guide: Management, Allocation, and Numericals

1. File Management: Basic Concepts

File System: Acts as the logical method for organizing, storing, naming, and accessing data on storage devices. It maps file data into logical blocks, manages directories, handles metadata, and allocates storage space to ensure efficient storage retrieval, security, and maintenance.

Key Concepts

  • File: The basic unit of storage, typically containing a name, extension, and data. It acts as a container for data (text, binary, executable).
  • Directory (Folder): A structure that stores file metadata and logical organization.
  • Attributes: Information about files (type, size, location, permissions, creation time).
  • Hierarchy: Files organized into a tree structure starting with a root directory.

Core Functions & Operations

  • File Mapping: Mapping logical files onto physical storage devices.
  • Access Control: Managing permissions to ensure data security.
  • Free Space Management: Tracking unused blocks to allocate for new files.

File Operations: Create/Delete, Open/Close, Read/Write, Reposition (Seek), and Truncating a file.

Detailed File Attributes

  • Name: Symbolic file name (only info kept in human-readable form).
  • Identifier: Unique tag (usually a number) identifying the file internally.
  • Type: Needed for systems supporting different file types.
  • Location: Pointer to a device and location of the file.
  • Size: Current size in bytes/words/blocks.
  • Protection: Access-control info (read/write/execute).
  • Timestamps: Creation, last modification, and last use times.
  • Extended Attributes: Character encoding, checksums.

Common File System Types

FAT (FAT32): Older format (Windows).
NTFS: Modern Windows, supports recovery.
ext (ext3, ext4): Commonly used in Linux.
HFS/APFS: Apple macOS and iOS.
XFS: Used in Linux/Unix.

Common File Types:

executable (exe, com), object (obj, o), source code (c, java), batch (bat, sh), markup (xml, html), word processor (docx, rtf), library (lib, dll), print/view (pdf, jpg), archive (zip, rar), multimedia (mp3, mp4).

2. File Access Methods

File access methods dictate how to read and write data stored in files on secondary devices.

A. Sequential Access

Data is accessed in a linear or sequential order, one after the other. Read commands move the pointer ahead by one. Write commands allocate memory and move pointer to the end. Reasonable for tape drives.

  • Advantages: Simple to implement. Less prone to data corruption. Fast for lexicographic access.
  • Disadvantages: Slow for searching specific records. Middle inserts/updates are difficult. Wastes storage for varying length records.

B. Direct Access Method

Allows data to be read/written directly at any block/record using its address. Random access without scanning previous records.

  • Advantages: Decreased average access time. Immediate access.
  • Disadvantages: Complex implementation. Higher storage overhead for maintaining pointers/address tables.

C. Indexed Access Method

Involves accessing files through an index/directory containing file names and locations. The Index Sequential Method is built on top of Sequential access, constructing an index with pointers to blocks.

  • Advantages: Fast searching, flexible, reduced access time for large files.
  • Disadvantages: High complexity, extra storage required for indexes, slower updates (both data and index must be updated).

3. Directory Structure

A directory translates file names into their file control blocks. It must allow insertions, deletions, searches, and listing.

  • Single-Level: All files contained in one directory. Pros: Easy to implement. Cons: Name collisions (no two files can have the same name), slow searching as it grows.
  • Two-Level: Each user has a User File Directory (UFD). A Master File Directory (MFD) points to UFDs, preventing conflicts between different users.
  • Tree-Structured: Most common in PCs. Upside-down tree with a root. Users can create nested subdirectories but cannot access other users' trees easily.
  • Acyclic-Graph: Allows a file/subdirectory to be shared across multiple directories using links. Changes by one user are visible to all. Strictly avoids loops.
  • General Graph: Can have cycles (loops back to starting points). Makes navigating and managing complex.

4. File-System Implementation

Refers to how the file system manages the storage and retrieval of data on physical devices.

Layers in File System

  1. Application Programs: Topmost layer for user interaction (CLI, browsers).
  2. Logical File System: Manages metadata. Maintains File Control Blocks (FCB).
  3. File Organization Module: Translates logical blocks to physical blocks. Tracks free space.
  4. Basic File System: Issues general commands to device drivers. Manages buffers/caches.
  5. I/O Control Level: Device drivers acting as hardware interface.
  6. Devices Layer: Actual hardware performing read/write.

Implementation Issues

  • Management of disk space (garbage collection, fragmentation).
  • Checking consistency and repairing errors (journaling, checksums).
  • Locking files and managing concurrency (semaphores).
  • Performance optimization (caching, prefetching).

5. Allocation Methods

Methods defining how files are stored in disk blocks.

A. Contiguous Allocation

Each file occupies a contiguous set of blocks. If it requires n blocks starting at b, it takes b, b+1... b+n-1.

Pros: Extremely fast (minimal seeks). Supports direct access (kth block is b+k).
Cons: Suffers from external and internal fragmentation. Hard to increase file size later.

B. Linked Allocation

File is a linked list of scattered disk blocks. Directory points to start and end blocks. Each block points to the next.

Pros: Very flexible file size. No external fragmentation.
Cons: Extremely slow for direct access (must traverse k blocks to find the kth block). Pointer overhead wastes space.

C. Indexed Allocation

A special Index block contains pointers to all blocks occupied by the file.

Pros: Fast direct access. No external fragmentation.
Cons: High pointer overhead. Wastes an entire block just for pointers for very small files.

6. Mass Storage Structure

The main mass-storage system of a computer is secondary storage (Hard Disk Drives and Non-Volatile Memory). Tertiary storage includes slower, larger capacity mediums like magnetic tapes and optical disks.

A. Hard Disk Drives (HDD) Physical Structure

HDDs are mechanical devices that store data magnetically. They consist of several moving parts:

Figure: Physical components of a Hard Disk Drive (HDD)
  • Platters: Flat, circular disks covered with magnetic material where data is physically written.
  • Read-Write Head: A mechanism that "flies" just above each platter surface to read or alter the magnetic polarity. They are attached to a single disk arm that moves all heads together as a unit.
  • Tracks: The surface of each platter is logically divided into concentric circular rings called tracks.
  • Sectors: Tracks are further subdivided into sectors. A sector is the fixed, smallest unit of data transfer (commonly 512 bytes).
  • Cylinder: The vertical collection of all matching tracks across all platters at a given arm position.
  • Spindle/Motor: Spins the platters at high speeds (RPMs like 5400, 7200, 10000, or 15000).

HDD Performance Metrics

Positioning Time (Random-Access Time): The total time required to reach a specific block of data. It is the sum of two factors:

  • Seek Time: The time it takes to mechanically move the disk arm and read-write head to the desired cylinder/track.
  • Rotational Latency: The time spent waiting for the desired sector to rotate under the read-write head.

B. Nonvolatile Memory Devices (NVM / SSD)

Unlike HDDs, NVMs (like Solid State Drives - SSDs, and USB flash drives) are entirely electrical, composed of a controller and flash NAND semiconductor chips.

  • No Moving Parts: Makes them drastically more reliable, durable, and shock-resistant.
  • Extreme Speed: They completely eliminate "Seek Time" and "Rotational Latency," making random data access near-instantaneous.
  • Connection: Modern NVMs connect directly to the system bus (e.g., PCIe interfaces) to maximize transfer rates.
  • Trade-offs: Historically more expensive per Gigabyte and generally have smaller total capacities compared to bulk HDDs, though the gap is closing.

7. I/O Hardware & Data Transfer Methods

I/O hardware consists of a system bus, a device controller, an I/O port, and its registers. To move data between the CPU/Memory and external devices, the operating system utilizes three primary transfer techniques.

1. Programmed I/O (PIO)

The CPU issues an I/O command and then repeatedly checks the device's status register to see if it is finished. This constant checking is called Polling or Busy Waiting.

Disadvantage: Severely wastes CPU cycles, as the processor is stuck doing nothing but asking "Are you done yet?" instead of executing other programs.

2. Interrupt-Driven I/O

The CPU issues an I/O command and immediately moves on to execute other processes. When the I/O device finishes its task, it sends a hardware Interrupt Signal to the CPU to grab its attention.

Advantage: Eliminates busy waiting. CPU time is utilized efficiently, making it ideal for slow character devices like keyboards.

3. Direct Memory Access (DMA)

Used for high-speed devices (like disks). The CPU offloads the work to a specialized DMA Controller (DMAC). The DMAC transfers massive blocks of data directly to/from main memory, completely bypassing the CPU.

Advantage: The CPU is only interrupted once at the very beginning (to set up the transfer) and once at the very end (when the entire block is successfully transferred).

8. Disk Organization

Before a raw physical disk can store files, the operating system must organize and structure it logically.

Key Steps in Disk Organization & Management

  • Logical Formatting (Clusters): The OS abstracts the physical hardware by grouping physical sectors into larger logical blocks (clusters) to improve read/write efficiency and manageability.
  • Partitioning: Dividing a single physical hard drive into multiple independent logical drives (e.g., C: drive, D: drive). This allows multiple file systems or different operating systems to coexist cleanly on the same disk.
  • High-Level Formatting: Preparing a partition by writing a specific File System structure (like NTFS, FAT32, or ext4) onto it, which dictates exactly how data directories will be indexed and stored.
  • The Boot Block: A special, reserved sector at the very beginning of the disk that stores the Bootstrap Loader—a tiny program responsible for starting the OS during computer power-up.
  • Bad Block Management: Disks inevitably develop physically defective sectors over time. The OS/Controller detects these "bad blocks" and logically maps them to hidden spare sectors so data is not corrupted.
  • RAID (Redundant Array of Independent Disks): A technology that combines multiple physical disks into a single logical unit to massively increase performance (via data striping) and reliability (via data mirroring/parity).
  • Common System Tools: Features like Windows Disk Management, Linux fdisk/parted, or macOS Disk Utility are used by administrators to perform these structural changes.

9. I/O Management

I/O Management is the OS subsystem responsible for coordinating the complex data flow between the CPU, main memory, and external peripheral devices.

Core Components & Responsibilities

  • Device Controllers vs. Device Drivers: The Device Controller is the physical electronic hardware component managing the actual device mechanics. The Device Driver is the OS software component that translates generic OS commands into specific hardware-level instructions the controller understands.
  • Buffering & Caching:
    • Buffering: Temporarily storing data in memory while it is being transferred to handle speed mismatches between extremely fast CPUs and relatively slow I/O devices.
    • Caching: Keeping copies of frequently accessed disk data in fast RAM to improve overall system read/write performance.
  • I/O Scheduling: Ordering and prioritizing read/write requests to optimize performance (e.g., reordering disk requests via algorithms like SCAN or SSTF to minimize mechanical head movement).
  • Interrupt Handling: The complex OS routines that pause the CPU, save its current state, and efficiently process the completion signals triggered by external devices.

Types of I/O Devices Managed

1. Block Devices

Store and transfer data in fixed-size blocks (e.g., 512 bytes or 4KB). They are capable of immediate random access.

Examples: Hard Drives, SSDs, Optical Drives.

2. Character Devices

Send and receive a continuous stream of characters (bytes) sequentially, one by one. They are not addressable for random access.

Examples: Keyboards, Mice, Serial Ports.

3. Network Devices

Specialized for remote communication over networks, transmitting data as discrete formatted packets.

Examples: Ethernet cards, Wi-Fi adapters.

10. Disk Scheduling Algorithms & Numericals

Goal: Minimize access time (seek time + rotational latency) by optimizing the order of read/write requests.

MASTER PROBLEM DATA FOR ALL ALGORITHMS:
Request Queue = 98, 183, 37, 122, 14, 124, 65, 67.
Initial Head Position = 53.

1. FCFS (First-Come, First-Served)

Serves requests in the exact order they arrive. Intrinsically fair, but generally not the fastest.

Path & Calculation:
53 → 98 → 183 → 37 → 122 → 14 → 124 → 65 → 67
Movements: |53-98|=45, |98-183|=85, |183-37|=146, |37-122|=85, |122-14|=108, |14-124|=110, |124-65|=59, |65-67|=2.
Total Head Movement: 640 cylinders.

2. SSTF (Shortest Seek Time First)

Selects the request closest to the current head position. Reduces total movement but can cause starvation for edge requests.

Path & Calculation:
53 → 65 → 67 → 37 → 14 → 98 → 122 → 124 → 183
Movements: |53-65|=12, |65-67|=2, |67-37|=30, |37-14|=23, |14-98|=84, |98-122|=24, |122-124|=2, |124-183|=59.
Total Head Movement: 236 cylinders.

3. SCAN (Elevator Algorithm)

Arm moves to one physical end of the disk, servicing along the way, reverses direction at the end, and continues. (Assumption: Total cylinders 0-199. Moving towards higher end 199 first).

Path & Calculation:
Sorted Queue: 14, 37, 65, 67, 98, 122, 124, 183.
Path: 53 → 65 → 67 → 98 → 122 → 124 → 183 → 199 (Disk End) → 37 → 14
Movements: (199 - 53) + (199 - 14) = 146 + 185
Total Head Movement: 331 cylinders.

4. C-SCAN (Circular SCAN)

Variant of SCAN for uniform wait time. Moves to one end, services requests, jumps immediately to the beginning (0) without servicing on the return trip, and sweeps again.

Path & Calculation:
Path: 53 → 65 → 67 → 98 → 122 → 124 → 183 → 199 (End) → 0 (Start) → 14 → 37
Movements: (199 - 53) + (199 - 0) + (37 - 0) = 146 + 199 + 37
Total Head Movement: 382 cylinders.

5. LOOK

Improved SCAN. Reverses direction immediately after the last request in a given direction, rather than going all the way to the physical edge of the disk.

Path & Calculation:
Move Higher: 53 → 65 → 67 → 98 → 122 → 124 → 183 (Last request, reverse!)
Move Lower: 183 → 37 → 14
Movements: 12 + 2 + 31 + 24 + 2 + 59 + 146 + 23
Total Head Movement: 299 cylinders.

6. C-LOOK

Improved C-SCAN. Jumps from the last request in one direction to the first request at the other end, bypassing the physical edges.

Path & Calculation:
Order: 53 → 65 → 67 → 98 → 122 → 124 → 183 (Jump!) → 14 → 37
Movements: |65-53|=12, |67-65|=2, |98-67|=31, |122-98|=24, |124-122|=2, |183-124|=59, |14-183|=169 (Wrap), |37-14|=23.
Total Head Movement: 326 cylinders.

Algorithm Comparisons (LOOK vs C-LOOK)

  • Direction: LOOK serves requests in both directions. C-LOOK serves in one direction only.
  • Performance: C-LOOK has the best overall performance among algorithms. LOOK lags behind C-LOOK.
  • Handling: C-LOOK handles requests more effectively than LOOK.
  • Variance: LOOK has higher throughput but C-LOOK provides perfectly uniform waiting and response times.
